# Requirements

Status: Draft (awaits human acceptance)

## Problem

用户需要一个极简的个人待办事项工具，在浏览器中使用，无需安装任何软件。同时作为 AIteam 工作流的第一个练手项目，跑通 BA → UX → Architecture → Development → QA → Review 完整流程。

## Users

- **唯一用户**：单个个人用户，无需多用户或登录。

## Scope

一个 Node.js 驱动的单页 Web 应用，功能如下：
- 查看待办列表
- 添加新待办
- 标记待办为已完成 / 未完成
- 删除待办

## User Stories And Acceptance Criteria

### US-1: 查看待办列表
- **AC1.1**: 打开页面即可看到当前所有待办事项，包括已完成和未完成。
- **AC1.2**: 未完成的待办显示为未勾选状态，已完成的显示为勾选 + 删除线。
- **AC1.3**: 列表为空时显示提示文字「暂无待办事项」。

### US-2: 添加待办
- **AC2.1**: 页面顶部有输入框和「添加」按钮。
- **AC2.2**: 输入文字后点击按钮（或按 Enter 键），新待办出现在列表顶部。
- **AC2.3**: 空内容不可添加（无反应或提示）。

### US-3: 标记完成 / 取消完成
- **AC3.1**: 每个待办左侧有勾选框，点击切换完成状态。
- **AC3.2**: 标记完成后，文字出现删除线效果。
- **AC3.3**: 取消完成后，文字恢复普通样式。

### US-4: 删除待办
- **AC4.1**: 每个待办右侧有删除按钮，点击后待办从列表中消失。
- **AC4.2**: 删除无需确认（极简设计）。

## Constraints

- **零外部依赖**：仅使用 Node.js 内置模块（`http`、`fs`、`path`）。
- **单文件服务端**：一个 `server.js` 文件。
- **内嵌前端**：HTML/CSS/JS 内嵌在服务端文件中，或不依赖构建工具的独立文件。
- **内存存储**：数据保存在服务端内存中（重启丢失，练手项目可接受）。
- **不低于 Node.js 18**。
- API 使用 RESTful 风格（JSON）。

## Assumptions

- 用户对页面视觉效果要求不高，极简可用即可。
- 数据不需要持久化（重启丢失可接受）。
- 浏览器为现代浏览器（Chrome/Firefox/Safari 近两年版本）。

## Open Questions

- 无（项目足够简单，待 UX 阶段细化视觉风格）。

## Out Of Scope

- 数据持久化（数据库/文件存储）
- 多用户 / 登录 / 权限
- 分类、标签、优先级、截止日期
- 移动端适配
- 测试框架（QA 阶段可做手动测试或简单脚本验证）
- 部署 / Docker
