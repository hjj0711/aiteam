# UX And UI Design

Status: Draft (awaits human acceptance)

## User Flow

```text
打开页面 → 看到待办列表（可能为空）
                │
                ├─ 输入文字 → 点击添加/按Enter → 新待办出现在列表顶部
                │
                ├─ 点击待办左侧复选框 → 切换完成/未完成 → 样式更新
                │
                └─ 点击待办右侧删除按钮 → 待办消失
```

单页面，无跳转，无弹窗。

## Screens And States

### 状态覆盖

| 状态 | 表现 |
|------|------|
| **空列表** | 列表区居中显示「暂无待办事项」灰色文字 |
| **有数据** | 显示待办列表 |
| **加载中** | 页面打开瞬间请求 API，列表区显示「加载中...」 |
| **请求失败** | 列表区显示「加载失败，请刷新重试」 |
| **添加成功** | 新待办从顶部出现，输入框清空 |
| **添加失败** | 输入框保留内容，底部短暂显示红色提示 |
| **删除/标记失败** | 操作回滚到之前状态，底部短暂显示红色提示 |

## Components

| 组件 | 描述 |
|------|------|
| **Header** | 页面标题「我的待办」，居中 |
| **InputGroup** | 输入框 +「添加」按钮，并排，按钮固定宽度 |
| **TodoList** | 承载所有 TodoItem，无数据时显示空状态 |
| **TodoItem** | 复选框 + 文字 + 删除按钮（✕），已完成项加删除线和灰色 |
| **Toast/ErrorBar** | 底部短暂出现的错误提示条（红色背景） |

## Responsive And Accessibility Requirements

- **宽度**：最大宽度 480px 居中，适配手机和桌面
- **颜色**：浅灰背景，白色卡片区域，可辨识的对比度
- **键盘操作**：Enter 提交输入，Tab 在输入框/复选框/删除按钮间移动
- **语义 HTML**：使用 `<main>`, `<form>`, `<ul>`, `<li>`, `<input type="checkbox">`, `<button>` 等标准标签
- **删除按钮**：有 `aria-label="删除 xxx"`
- **完成项文字**：用 `<s>` 标签或 `text-decoration: line-through`，配合 `aria-checked`
